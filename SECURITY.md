# Security Policy

## Supported Scope

This project focuses on small CLI binaries that may run in automated agent environments. Security issues that affect command execution, filesystem access, environment handling, or output integrity are in scope.

## Reporting

Please do not open a public issue for a suspected security vulnerability.

Instead:

1. Use GitHub security reporting if available for this repository.
2. If that is not available, contact the maintainers privately through GitHub.
3. Include reproduction steps, affected versions or commits, and impact.

## Response Goals

Maintainers will aim to:

- acknowledge the report quickly
- confirm impact and scope
- ship a fix or mitigation when warranted
- credit reporters unless they prefer otherwise
